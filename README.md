# 🐱 CatTarven — 猫猫酒馆

一款基于 Android 原生 Jetpack Compose 构建的角色扮演 (RP) AI 对话客户端。兼容 OpenAI API 格式的任意大模型后端，完整支持 SillyTavern V2/V3 角色卡导入导出。面向对 Token 效率、上文管理和沉浸感有极致追求的硬核 RP 玩家设计。

> **最低系统要求**：Android 12 (API 31)  
> **目标 SDK**：Android 15 (API 36)

## 🚀 快速开始

1. 打开应用，点击右上角 **⚙ 设置**。
2. 在 **API 连接** 区域填写：
   - **API URL**：你的 OpenAI 兼容 API 基础地址（例如 `https://api.openai.com` 或任何中转站地址）。
   - **API Key**：你的密钥。
   - **Model Name**：要调用的模型名称（例如 `gpt-4o`、`gemini-2.5-pro` 等）。
3. 点击 **"测试连接"** 按钮验证。成功后，下方会展示服务端所有可用模型列表，**点击模型名称可一键选择**，并且列表会自动滚动到你已选中的模型位置。
4. 返回主页，点击右下角 **"+"** 创建一个角色，或者点击右上角 **导入按钮** 导入现有的角色卡文件。
5. 点击角色卡进入聊天，开始角色扮演。

## ✨ 核心功能

### 1. 角色卡管理

#### 创建角色
在角色编辑页面可以配置以下字段：
| 字段 | 说明 |
|---|---|
| 角色名称 | **必填**。同时作为聊天气泡的显示名称。 |
| 角色描述 | 角色的外貌、身份、背景故事。会作为系统消息发送给大模型。 |
| 开场白 | 角色在新对话中自动发送的第一条消息。支持 `{{user}}`、`{{char}}` 等宏变量。 |
| 角色系统提示词 | 角色专属的 System Prompt，追加在全局提示词之后。 |
| 创作者备注 | 仅供备注参考，不会发送给 AI。 |
| 角色头像 | 点击头像区域即可从相册选择。 |
| 对话背景 | 可以为每个角色设置独立的聊天背景图片和模糊度，未设置则使用全局背景。 |

#### 导入角色卡
在主页点击右上角导入按钮，支持以下格式：
- **SillyTavern V2 / V3 JSON 格式**（`spec: "chara_card_v2"` / `"chara_card_v3"`）
- **SillyTavern V1 扁平 JSON 格式**（根层 `first_mes`、`character_book` 等字段）
- **PNG 图片角色卡**：自动从 PNG 的 `tEXt` 块中提取 Base64 编码的 `chara` 数据（SillyTavern 标准），并同时将该 PNG 设为角色头像。
- **简单 Map 格式**：仅包含 `name`、`description` 等基础键值对的 JSON。

导入时会自动解析并合并：
- 内嵌的 `character_book`（世界书词条）
- `extensions.regex_scripts`（角色专属正则脚本）
- `alternate_greetings`（备用开场白）

#### 导出角色卡
在角色编辑页面点击右上角导出按钮，支持两种格式：
- **📄 导出为 JSON**：标准 SillyTavern V2 格式，包含世界书条目、正则脚本等全部数据。
- **🖼️ 导出为 PNG**：将角色数据以 Base64 编码嵌入 PNG `tEXt` 块。如果角色有头像则使用头像，否则自动生成一张渐变背景 + 首字母的默认头像。

#### 管理操作
在主页长按角色卡片，弹出操作菜单：
- **编辑角色卡** — 进入角色编辑页面
- **新建对话** — 为该角色新开一个空白对话存档
- **导出角色卡 (JSON)** — 快速导出
- **导出当前对话记录 (TXT)** — 将当前对话的所有消息以 `名字:\n内容` 格式导出为纯文本文件
- **删除角色卡** — 删除角色及其所有对话存档
- **删除当前对话** — 仅删除当前对话存档，保留角色
- **取消**

主页顶部有**搜索栏**，支持按角色名称和描述实时模糊搜索。

### 2. 世界书系统 (World Info)

在聊天页面顶栏的 **"📖"** 按钮即可打开世界书管理面板。

#### 词条类型
每个词条有三种**标签类型**（tag），决定其在上文中的插入位置：
| 标签 | 说明 |
|---|---|
| `normal`（普通条目） | 根据"相对位置"设定，插入在主要设定的前方或后方。 |
| `main_setting`（主要设定） | 作为主体设定（如角色的核心描述），在上文中只取第一条。 |
| `writing_rules`（写作规则） | 插入在所有历史对话之后，作为最终的写作指导。 |

#### 触发方式
- **常驻 (Constant)**：无论是否命中关键词都始终注入上文。
- **关键词触发**：系统会在最近 10 条对话中搜索词条的关键词列表，命中任一关键词即激活该词条。
- **禁用 (Disable)**：手动关闭该词条，不参与任何匹配。

#### 消息身份 (Role)
每个词条可以自定义作为 `system`、`user` 或 `assistant` 角色注入上文。

#### 导入外部世界书
在世界书面板底部点击 **"📂 导入世界书 JSON"**，支持 SillyTavern 标准格式：
- 字典格式 `{"entries": {"0": {...}, "1": {...}}}`
- 列表格式 `{"entries": [{...}, {...}]}`

兼容字段名差异（如 `key` / `keys`、`insertion_order` / `insertionorder`、`secondary_keys` / `keysecondary`）。

### 3. 聊天界面

#### 富文本渲染
聊天内容在 Android WebView 中使用 `marked.js` 和 `highlight.js` 渲染，支持：
- **Markdown**：加粗、斜体、删除线、引用块、有序/无序列表
- **代码块**：带语法高亮（JavaScript、Python、Kotlin、Bash、C/C++、Java、JSON、Markdown 等数十种语言）
- **HTML 标签**：可在设置中开关"富文本 (HTML) 渲染"
- **表格** 渲染
- **深色 / 浅色主题**：跟随全局设置自动切换

#### 消息操作
长按任意聊天气泡弹出操作菜单（通过 WebView Bridge 与原生层通信）：
- **📋 复制** — 复制消息原始文本（正则替换前的内容）
- **✏️ 编辑** — 弹出编辑弹窗，修改消息内容后实时更新
- **🔄 重新生成** — 为该消息追加一个新的 Swipe 版本（见下文 Swipe 系统）
- **🗑️ 删除** — 删除该消息

#### 思维链 (Reasoning) 支持
如果模型返回 `reasoning_content` 字段（如 DeepSeek-R1、QwQ 等思维链模型），或在回复中包含 `<think>...</think>` 标签：
- 思维过程会被自动分离，以可折叠的蓝色区块 **"💭 查看推理过程"** 展示
- 正文内容不受 think 标签影响

#### 输入框草稿持久化
输入框中正在编写但尚未发送的文字，会在切换页面（如进入设置、编辑角色）后自动保留。只要是同一个对话，草稿不会因为导航而丢失。

#### 前台通知
AI 生成回复时，应用会启动前台服务并**立即**弹出通知栏提示 "正在生成回复，请稍候..."，防止应用被系统后台杀掉。生成完成后自动关闭通知。

### 4. 多模态附件发送

在聊天输入框左侧的 **📎 附件按钮**，点击可打开系统文件选择器上传文件。

#### 支持类型
| 类型 | 处理方式 |
|---|---|
| **图片**（jpg, png, webp 等） | 自动在本地进行智能压缩（最大 1024×1024、JPEG 80% 画质），转换为 Base64 编码，以 OpenAI 多模态格式 `image_url` 发送。 |
| **文本文件**（txt 等） | 自动读取文件内容，以 `[用户上传了文件: xxx.txt]\n文件内容:\n...` 格式拼入消息正文。 |

#### 使用方式
1. 点击附件按钮选择文件
2. 选中的文件以缩略图形式悬浮在输入框上方（图片显示预览图，文本显示文件图标和文件名）
3. 点击缩略图右上角的 ❌ 可移除
4. 输入框可以同时输入文字，也可以留空
5. 点击发送按钮一起发出

#### 限制
- 单次最多上传 **3 张图片**
- 图片在发送前自动压缩，避免 `HTTP 413 Payload Too Large` 错误
- 需要后端模型支持视觉能力（Vision），不支持的模型会返回 API 错误，应用不会闪退

#### 聊天气泡展示
- 发送的图片会在聊天气泡中显示为缩略图，**点击可放大查看原图**
- 文本文件显示为带圆角的标签样式

### 5. Swipe 多版本回复系统

#### 基本功能
每条 AI 回复底部都有 **左右箭头按钮**，可以浏览该位置的多个回复版本。中间显示当前页码 `1/3`。

#### 自动多版本生成 (Auto-Swipe)
在设置中调节 **"每次生成备选回复数"**（1~5），发送消息后：
- **非流式模式**：一次请求让 API 返回 N 个 `choices`（利用 `n` 参数），自动生成 N 个 Swipe 版本。
- **流式模式**：发出 1 次流式请求。

#### 手动重新生成
在任意 AI 消息上长按选择 "重新生成"，会在该消息的 Swipe 列表末尾**追加**一个新版本，而非替换当前版本。所有历史版本均保留。

#### 开场白 Swipe
如果角色卡配置了 `alternate_greetings`（备用开场白），首条 AI 消息支持在多个开场白之间左右 Swipe 切换。

### 6. 正则脚本扩展 (Regex Extension)

#### 两级正则
1. **全局正则**：在设置页 → "正则脚本扩展" 中管理，对所有角色生效。
2. **角色专属正则**：在角色编辑页中管理，仅对该角色生效。导入 SillyTavern 角色卡时会自动解析 `extensions.regex_scripts`。

#### 工作原理
- 正则仅作用于**前端显示层**（WebView JavaScript），不会修改存储的原始消息。
- 发送给 API 的上文使用的是**原始未替换的文本**。
- 每条规则包含：名称、正则模式 (Pattern)、替换内容 (Replacement)、启用/禁用开关。
- 使用 JavaScript `new RegExp(pattern, 'gs')` 执行，支持全局匹配和跨行。

### 7. 上文状态栏智能剥离

**位置**：设置页 → **"🧹 上文状态栏剥离"**

#### 使用场景
很多角色卡会在设定中要求 AI 每次回复时附带一段"状态栏"（如角色生命值、好感度、当前场景等）。这些状态栏在每条历史消息中反复出现，浪费大量 Token 且可能干扰 AI 的连贯性。

#### 使用方式
1. 在输入框中填入一个正则表达式来匹配你的状态栏格式。
2. **留空**则不启用。

#### 工作原理
- 每次向 API 构建消息时，会对**除最后一条 AI 回复以外**的所有历史 AI 消息应用该正则，删除匹配到的内容。
- **最后一条 AI 回复**保持完整发送，确保 AI 知道当前的最新状态。
- **仅影响发送给 API 的内容**，不会修改聊天记录的显示。

#### 常用正则示例
| 状态栏格式 | 正则表达式 |
|---|---|
| `<status>...</status>` | `<status>[\s\S]*?</status>` |
| `【状态栏】...【/状态栏】` | `【状态栏】[\s\S]*?【/状态栏】` |
| `---\n状态:...(到文末)` | `---\n状态:[\s\S]*$` |

### 8. 快捷道具系统 (Props)

**位置**：设置页 → **"🎒 快捷道具管理"**

预设常用的长文本内容（如"请总结当前剧情"、"检查你的行为是否合理"等），在聊天界面点击**道具按钮 🎒** 即可一键发送，免去重复打字。

每个道具有**名称**和**发送内容**两个字段。发出的消息在气泡顶部会带有金色道具名称标签。

### 9. 实验室 (Lab)

**入口**：主页右上角 **🧪 按钮**

每次发送消息给 AI 时，应用会自动保存完整的 API 请求 JSON 到实验室。

#### 功能
- 按时间倒序展示所有请求记录，显示角色名称、时间戳和 JSON 大小
- 点击记录可查看完整的 JSON 内容
- 支持**一键复制**到剪贴板
- 支持**一键清空**所有日志

#### 性能优化
包含图片的请求中，Base64 编码字符串会在保存前被自动替换为 `<图片 Base64 编码已隐藏，长度: xxxxx>` 标记，防止巨大的 JSON 字符串导致查看时界面卡死。

### 10. 转生功能

**位置**：聊天页面菜单 → **"一键转生"**

一键将当前对话的内容总结发送到一个全新的对话存档中，重新开始角色扮演的同时延续核心记忆。操作流程：
1. 输入你想让 AI 延续的总结或提示文本
2. 系统自动创建新对话、插入角色开场白、然后以你的输入作为第一条用户消息发出
3. AI 在新对话中生成回复

### 11. API 参数与预设管理

## 📐 上文构建顺序

应用向 API 发送的消息数组严格按以下 **八层顺序** 构建：

```
① 全局系统提示词 (system)
② 角色系统提示词 (system)
③ 玩家设定/马甲 (system)
④ 主要设定前的世界书条目 (各自 role)
⑤ 主要设定条目 (其 role)
⑥ 主要设定后的世界书条目 (各自 role)
⑦ 历史对话消息 (交替 user/assistant)
⑧ 写作规则条目 (其 role)
```

世界书的关键词匹配范围为最近 **10 条**消息的拼接文本。

发送消息中的角色 `name` 字段会自动进行合法化处理（替换中文等非 `[a-zA-Z0-9_-]` 字符为空），防止 API 返回 400 错误。

## 🏗️ 项目架构

采用 **MVVM (Model-View-ViewModel)** 架构：

```
┌─────────────────────────────────────────────┐
│                    UI Layer                  │
│  Screens (Compose)  ←→  Components (WebView) │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│              ViewModel Layer                 │
│  ChatViewModel  CharacterViewModel           │
│  SettingsViewModel  LabViewModel             │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│               Data Layer                     │
│  Repository (Chat, Character, Settings, Lab) │
│  API Service (OpenAI SSE/REST)               │
│  Model (Character, ChatMessage, WorldInfo)   │
└─────────────────────────────────────────────┘
```

- **聊天渲染**：Compose 原生 UI + 嵌入式 WebView（通过 `@JavascriptInterface` Bridge 双向通信）
- **数据持久化**：角色和对话以独立 JSON 文件存储在 `filesDir`；设置使用 `SharedPreferences`
- **网络层**：OkHttp 直接处理 HTTP 请求和 SSE 流解析，不依赖 Retrofit
